# Feature Specification: bscp MCP 支持文件型配置

**Feature Directory**: `specs/stories/135633598`
**Created**: 2026-07-02
**Status**: Draft
**Input**: 需求 135633598 —— 为 bscp 文件型配置（config_type=file）补齐 AI 对话式操作能力（MCP + skill），并新增「仅返回下载 URL」的管理面接口，避免大文件内容穿透管理面/网关造成带宽负载。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 文件型配置现状查询 (Priority: P1)

作为配置管理者/SRE 或 AI Agent，我希望通过对话（MCP + skill）查询某个文件型服务的配置项与已发布内容，从而在不登录 UI 的情况下快速掌握配置现状。

**Why this priority**：查询是所有变更操作的前置。没有可靠的查询能力，后续增删改、发布与验证都无从定位对象；且查询为只读、风险最低，可独立交付价值。

**Independent Test**：对一个 `config_type=file` 的服务发起查询，返回配置项元数据（含内容 sign、byte_size、path、name）与已发布列表；对非 file 型服务发起查询时明确报错并中止。即可验证本故事独立可用。

**Acceptance Scenarios**：

1. **Given** 一个 `config_type=file` 的服务，**When** 通过 MCP+skill 查询其草稿态配置项与已发布内容，**Then** 返回配置项元数据（含 sign / byte_size / path / name）与已发布配置项列表。
2. **Given** 一个非 file 型（如 KV 型）服务，**When** 通过 MCP+skill 发起文件型查询/操作，**Then** 明确报错并中止，不返回文件型结果。
3. **Given** 目标服务不存在，**When** 发起查询，**Then** 返回明确的「服务不存在」提示并停止。

---

### User Story 2 - 文件型配置变更并发布生效 (Priority: P1)

作为配置管理者/SRE 或 AI Agent，我希望通过对话对文件型配置项做元数据增删改（引用已上传内容的 sign），再生成版本并全量发布，从而完成一次文件型配置变更闭环。

**Why this priority**：变更 + 发布是文件型配置管理的核心闭环，直接对应用户「完成一次配置变更」的主诉求；全量发布是最常用的生效路径。

**Independent Test**：基于已上传内容的 sign 创建/更新/删除配置项，确认变更处于草稿态（客户端不可见）；随后生成版本得到 releaseId 并全量发布，确认已发布内容可查询到。

**Acceptance Scenarios**：

1. **Given** 已上传内容对应的 sign，**When** 通过 MCP+skill 创建/更新/删除文件型配置项（仅元数据），**Then** 变更成功且处于草稿态，客户端不可见。
2. **Given** 存在草稿态改动，**When** 依次执行「生成版本 → 全量发布」，**Then** 生成 releaseId 且全量发布成功，已发布内容可被查询到。
3. **Given** 创建/更新配置项时引用的内容 sign 尚未上传，**When** 发起该变更，**Then** 返回明确的「内容未上传」错误并引导用户走 UI/SDK 上传，不产生指向空对象的引用。
4. **Given** 目标版本已有版本上线中或版本已废弃，**When** 执行全量发布，**Then** 按报错对照给出明确处置提示。

---

### User Story 3 - 通过下载 URL 验证发布结果（内容不穿透管理面） (Priority: P1)

作为配置管理者/SRE 或 AI Agent，我希望在发布后拿到文件的**下载 URL**（而非文件内容本身），并据此直连存储下载验证发布结果，从而避免大文件内容穿透管理面/网关造成带宽负载。

**Why this priority**：这是本需求的核心新增接口，也是「性能痛点」的直接解法；发布后的验证闭环依赖它。它与现有直接返回内容的下载接口并列，独立可测。

**Independent Test**：对一个已发布文件配置项的内容 sign 调用新增的「下载 URL」接口，确认响应体只含临时预签名下载 URL 与有效期、不含文件字节；用该 URL 可成功下载到正确文件。

**Acceptance Scenarios**：

1. **Given** 一个已发布文件配置项的内容 sign，**When** 调用新增的「下载 URL」接口，**Then** 返回临时预签名下载 URL 与有效期，且响应体**不包含**文件字节；用该 URL 可成功下载到正确文件。
2. **Given** 存储处于多副本（ha）模式返回多条预签名 URL，**When** 调用下载 URL 接口，**Then** 返回体的下载 URL 取首个有效 URL，且有效期为 3600 秒。
3. **Given** 传入的内容 sign 未上传，**When** 调用下载 URL 接口，**Then** 返回明确的「内容未上传」错误，不产生指向空对象的 URL。
4. **Given** 调用方未通过业务/服务（内容）鉴权，**When** 调用下载 URL 接口，**Then** 鉴权失败并返回预期错误，不泄露下载 URL。

---

### User Story 4 - AI 对话式操作能力可用（skill + MCP 工具可见） (Priority: P1)

作为 AI Agent，我希望有一份文件型配置的 skill 指引并且新接口已纳入自动生成的 MCP 工具集，从而正确编排调用顺序、规避字段级业务约束与常见报错，完成端到端对话式操作。

**Why this priority**：skill 是把上述接口能力对外暴露为可对话操作的载体；下载 URL 接口须在网关注册后才会进入 MCP 工具集供 skill 调用。二者是「AI 可用」的必要条件。

**Independent Test**：加载 `bk-bscp-prod-server-mcp`，确认新下载 URL 接口对应工具出现在工具集合中；按 skill 指引可完成「查询 → 元数据增删改 → 生成版本 → 发布 → 取下载 URL 验证」的端到端编排。

**Acceptance Scenarios**：

1. **Given** 新增的下载 URL 接口已在网关注册，**When** 加载 `bk-bscp-prod-server-mcp`，**Then** 该接口对应工具出现在 MCP 工具集合中，可被 skill 调用。
2. **Given** 文件型配置 skill 已提供，**When** AI 依据 skill 编排 F-001~F-006 的调用顺序，**Then** 能正确区分文件型与 KV 型差异（内容 sign + 元数据组织、草稿态需生成版本+发布后生效），并按报错→处置指引处理常见错误。

---

### User Story 5 - 灰度发布 (Priority: P2)

作为配置管理者/SRE 或 AI Agent，我希望能对文件型配置执行灰度发布（按分组或按标签），从而在小范围先行验证后再全量放开。

**Why this priority**：灰度是进阶发布能力（需求标注为 P1「应该有」，相对全量发布优先级次之），可在核心闭环稳定后独立交付。

**Independent Test**：对一个已生成的 releaseId 执行灰度发布（按分组/标签，all=false），确认仅目标灰度范围生效、非目标范围不受影响。

**Acceptance Scenarios**：

1. **Given** 一个已生成的 releaseId，**When** 执行灰度发布（按分组或按标签，all=false），**Then** 仅目标灰度范围生效。
2. **Given** 灰度按分组发布且 groups 有值，**When** 提交发布，**Then** all 必须为 false，否则按规则报错。

---

### Edge Cases

- **非 file 型服务**：对 KV 型或其它类型服务发起文件型操作时，必须报错并中止（R-002）。
- **内容 sign 未上传**：创建/更新配置项或调用下载 URL 接口时引用的 sign 不存在，需明确报错并引导走 UI/SDK 上传，不得产生指向空对象的引用/URL（Q-002 / AC-T01）。
- **草稿未发布即查询已发布**：草稿态改动在「生成版本 + 发布」前对客户端不可见，查询已发布内容不应看到未发布改动（R-001）。
- **发布冲突**：目标已有版本上线中或版本已废弃时，全量/灰度发布须给出明确处置提示。
- **灰度参数冲突**：groups 有值时 all 必须为 false（R-003），否则报错。
- **多副本存储返回多条 URL**：下载 URL 接口需确定性地取首个有效 URL（TR-001 / AC-T02）。
- **下载 URL 到期**：返回的预签名 URL 为临时 URL，超过有效期后失效，不可复用。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**（F-001）：系统必须支持查询文件型服务的配置项列表/详情与已发布配置项，返回内容 sign、byte_size、path、name 等元数据；查询前须校验服务为 `config_type=file`。
- **FR-002**（F-002）：系统必须支持对文件型配置项做元数据增删改（创建/更新/删除/批量 upsert），且创建/更新时引用的内容 sign 须为已上传内容；变更结果为草稿态，客户端不可见。
- **FR-003**（F-002 约束）：当引用的内容 sign 未上传时，系统必须返回明确的「内容未上传」错误并引导用户走 UI/SDK 上传，不得产生指向空对象的引用（本期不含内容上传编排）。
- **FR-004**（F-003）：系统必须支持基于草稿态改动生成不可变版本（CreateRelease），并返回 releaseId。
- **FR-005**（F-004）：系统必须支持对指定 releaseId 执行全量发布（all=true），发布成功后已发布内容可被查询到。
- **FR-006**（F-005）：系统应支持灰度发布（按分组或按标签，all=false + 灰度参数），仅目标灰度范围生效；groups 有值时 all 必须为 false。
- **FR-007**（F-006，核心）：系统必须新增一个管理面接口，输入内容 sign（及业务/服务或模板空间归属），只返回该内容的**临时预签名下载 URL 与有效期**，**不返回文件内容本身**。
- **FR-008**（F-006）：下载 URL 接口返回体须包含下载 URL 与有效期；当底层存储返回多条 URL 时，取首个有效 URL；有效期沿用存储层现有 3600 秒。
- **FR-009**（F-006 安全）：下载 URL 接口须与现有内容下载接口一致地做业务级 + 服务/内容级鉴权；鉴权失败不得返回下载 URL。
- **FR-010**（兼容性）：新增下载 URL 接口不得改变现有「返回文件内容」的下载接口行为，老接口保留，二者并列。
- **FR-011**（F-008）：下载 URL 接口须在蓝鲸 API 网关注册，使其自动纳入 `bk-bscp-prod-server-mcp` 生成的工具集合，供 skill 调用。
- **FR-012**（F-007）：系统必须提供一份文件型配置 skill，对标现有 KV skill，覆盖领域模型、端到端调用编排（F-001~F-006）、参数获取、字段级业务约束（R-001~R-004）、报错→处置与场景化示例；明确文件型与 KV 型差异，不含内容上传编排。

### Business Rules

- **R-001**：文件型配置项增删改均为草稿态，客户端不可见，必须「生成版本 + 发布」后才生效。
- **R-002**：文件型操作只适用于 `config_type=file` 的 app；对 KV 型执行文件接口应报错。
- **R-003**：灰度发布字段（灰度模式 / groups / labels）在发布动作上；groups 有值时 all 必须为 false。
- **R-004**：本期文件内容为只读引用——创建/更新配置项引用的内容 sign 须为已上传内容；上传动作不在本期范围。

### Out of Scope（本期不包含）

- 文件内容**上传**（仍走 UI / SDK；AI 场景不上传二进制大文件）。
- 模板型（template）文件配置的操作。
- KV 型能力的任何改动。

### Key Entities

- **biz（业务）**：配置操作的顶层归属，携带 bizId。
- **app（服务）**：`config_type=file` 的服务，是文件型配置项的容器；文件型操作只在此类 app 上有效。
- **config_item（配置项）**：草稿态的文件型配置项，由「内容 sign（SHA256）+ 元数据（path、name、byte_size、权限等）」组织。
- **release（版本）**：由草稿态改动生成的不可变版本快照，携带 releaseId，是发布的对象。
- **content（内容对象）**：以 sign 标识、已上传到对象存储的文件内容；本期只读引用，不在本期上传。
- **download URL（下载 URL）**：针对某内容 sign 生成的临时预签名下载 URL，携带有效期，到期失效，用于替代内容直传。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**：配置管理者/AI 无需登录 UI，仅通过对话即可查询到文件型服务的配置项元数据与已发布内容；对非 file 型服务操作 100% 被拦截并明确报错。（覆盖 AC-001）
- **SC-002**：配置管理者/AI 仅通过对话即可完成「元数据增删改 → 生成版本 → 全量发布」的完整变更闭环，发布后已发布内容可查询确认。（覆盖 AC-002 / AC-003）
- **SC-003**：配置管理者/AI 可对已生成版本执行灰度发布，且效果范围严格限定在目标分组/标签内，非目标范围不受影响。（覆盖 AC-004）
- **SC-004**：发布验证链路中，管理面/网关**不透传任何文件字节**——下载 URL 接口响应体不含文件内容；调用方凭返回的临时 URL 可成功下载到正确文件。（覆盖 AC-005 / AC-P01）
- **SC-005**：下载 URL 接口的鉴权与现有内容下载一致（业务 + 服务/内容级）；未授权调用无法取得下载 URL；返回的 URL 为临时 URL，超过 3600 秒后失效。（覆盖 AC-S01 / AC-T02）
- **SC-006**：内容 sign 未上传时，增删改与下载 URL 调用 100% 返回明确的「内容未上传」错误并引导走 UI/SDK 上传，不产生指向空对象的引用/URL。（覆盖 AC-T01）
- **SC-007**：下载 URL 接口在网关注册后，其对应工具出现在 `bk-bscp-prod-server-mcp` 工具集合中并可被 skill 调用；AI 依据文件型 skill 可独立完成端到端编排。（覆盖 AC-006）

## Assumptions

- 目标服务的 `config_type` 可通过现有服务查询能力可靠辨别 file / KV 型（沿用现有服务查询接口）。
- 被引用的文件内容此前已经由 UI/SDK 上传到对象存储，且其 sign（SHA256）可获取；本期不承担上传职责。
- 下载 URL 的有效期沿用存储层现有固定值 3600 秒，本期不新增有效期配置项（Q-003 已确认）。
- 下载 URL 接口落在管理面（api-server），与现有内容下载接口并列，复用同一鉴权链与存储层预签名能力（Q-004 已确认）；返回体命名为 `download_url` 与 `expire_seconds`（字段以实现为准）。
- 网关注册与 MCP 工具生成沿用现有注册脚本与文档生成链路，新接口注册后自动纳入工具集（F-008）。
- 文件型 skill 为文档产物，对标现有 `bscp-kv-config` skill 的结构与深度。
- 优先级标注（Q-001）为项目管理属性，不影响本规范的技术范围与验收，由 PO 在 TAPD 侧补充。
