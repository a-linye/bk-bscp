### 详细功能描述

#### [F-001] 验证需求澄清阶段

- **输入**：TAPD 需求 ID（本需求 1020451610135491554）
- **处理逻辑**：
  1. 通过 TAPD MCP 读取需求详情
  2. 结合背景知识进行多维度澄清并与用户确认
  3. 生成规范化需求文档并保存到 `docs/reqs/`
  4. 将文档内容回写 TAPD 需求 description 字段
- **输出**：本地规范化需求文档 + TAPD 需求描述更新成功
- **边界条件**：
  - 本阶段**仅回写需求描述**，不写入工时与规模数据
- **异常处理**：
  - TAPD 回写失败 → 重试一次，仍失败则输出文档供手动更新
