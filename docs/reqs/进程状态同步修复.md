## 需求背景

### 业务背景

BSCP 周期性地将进程实例的运行状态从 GSE 同步回本地，用于在控制台展示进程是否运行、是否被托管。当前同步流程（`internal/processor/gse/sync_gse.go` 的 `SyncSingleBiz`）会拉取业务下所有「拥有进程实例」的进程，逐个调用 GSE 查询状态。

存在以下问题：当某进程所在主机的 agent 处于异常（abnormal）状态时，GSE 对该进程的状态查询长期返回 `ErrorCode==115`（仍在执行中 / in-progress）。同步逻辑会按 `MaxInProgressRetries=5` 次、每次间隔 `DefaultInterval=3s` 重试（约 15s `defaultMaxWait`）后才放弃。当业务下存在较多 agent 异常的进程时，这些进程会持续占用同步并发额度并耗尽等待时间，拖慢甚至拖垮整批状态同步，最终导致同步超时。

agent 异常的主机本身就无法返回真实的进程状态，对其执行状态查询没有意义。因此需要在同步前将 agent 状态为 abnormal 的进程剔除，不参与状态同步。

### 用户故事

作为 BSCP 进程状态同步的运维/使用方
我想要 在周期性进程状态同步时自动跳过 agent 异常的进程
以便于 避免异常 agent 的 115 状态拖慢整批同步、消除同步超时

## 功能需求

### 核心功能点

| 功能编号 | 功能描述 | 优先级 | 涉及角色 | 备注 |
|---------|---------|--------|---------|------|
| F-001 | 周期性批量状态同步剔除 agent 异常进程 | P0 | 后台同步任务 | 必须 |

### 详细功能描述

#### [F-001] 周期性批量状态同步剔除 agent 异常进程

- **输入**：周期性同步任务触发 `SyncSingleBiz`，针对单个业务执行进程状态同步。
- **处理逻辑**：
  1. 拉取业务下待同步进程时，依据进程当前存储的 `agent_status` 字段过滤。
  2. `agent_status == abnormal` 的进程不纳入本轮 GSE 状态查询。
  3. 仅对 `agent_status == normal`（即非 abnormal）的进程执行原有的 GSE 状态查询与回写流程。
- **输出**：本轮同步仅更新非异常进程的实例状态；异常 agent 进程被跳过。
- **边界条件**：
  - 过滤后无可同步进程时，同步流程正常结束（无更新，不报错）。
  - agent 状态来源为数据库中存储的 `agent_status`（由 CMDB 同步流程周期性写入），不在本同步流程中实时查询 GSE agent 状态。
  - agent 由 abnormal 恢复为 normal 后，依赖 CMDB 同步更新 `agent_status`，下一轮状态同步即自动重新纳入该进程，无需额外处理。
- **异常处理**：
  - 被剔除（agent 异常）的进程，其进程实例 `process_instance` 状态字段保持上一次同步的值不变，本轮不写库。
  - 被剔除进程的 `process_state_synced_at`（进程状态同步时间）不更新，保留上一次成功同步的时间。

## 非功能需求

### 性能需求

- **同步耗时**：消除因 agent 异常进程反复触发 `ErrorCode==115` 重试（单进程最长约 15s）导致的同步超时；批量同步整体耗时不再随异常 agent 数量线性恶化。

### 安全需求

- 无新增数据权限/加密要求，沿用既有同步链路的租户与业务隔离。

### 可用性与稳定性

- 过滤逻辑不得影响正常 agent 进程的状态同步结果。
- agent 状态来源的延迟由 CMDB 同步周期决定，属可接受的最终一致。

### 兼容性

- 不改变 GSE 接口契约与现有数据表结构（复用已存在的 `agent_status` 字段）。

## 业务规则

### 业务逻辑规则

- **规则 R-001**：仅 `agent_status == abnormal` 的进程被排除出周期性状态同步；其余状态（normal）正常同步。
- **规则 R-002**：agent 状态判断以 DB 中存储值为准，不在状态同步流程内实时探测。

### 数据校验规则

- 复用既有 `AgentStatus` 枚举（`abnormal` / `normal`），不新增取值。

### 权限规则

- 沿用现有后台同步任务的权限模型，无新增。

## 边界范围

### 本期包含

- 周期性批量状态同步 `SyncSingleBiz` 中按 `agent_status` 剔除 abnormal 进程。

### 本期不包含

- 用户手动触发的单进程状态同步 `SyncSingleProcessStatus`（`process_state_sync` 任务）不在本期改造范围。
- 不在状态同步流程内实时查询/刷新 agent 状态。
- 不调整 `MaxInProgressRetries` / `defaultMaxWait` 等 115 重试参数。
- 不变更 agent 异常进程实例状态的展示语义（仍展示上次同步值）。
