# TAPD 需求评分

## 概述

对需求进行工时预估和 RICE 价值规模评分，并将评分结果更新到需求文档中。评分基于
需求文档的详细内容，从高级工程师视角进行客观估算。

## 输入

| 参数 | 来源 | 说明 |
|------|------|------|
| 需求文档 | 主 skill 传入或本地文件路径 | 规范化的需求文档（Markdown） |
| 背景知识 | 主 skill 传入 | 项目技术栈、架构等参考信息 |

## 执行流程

### 1. 加载与检查

加载需求文档内容，检查以下内容：
- 文档中是否已存在"预估工时"字段且有值？
- 文档中是否已存在"价值规模"（RICE 评分）且有值？
- 用户是否明确要求重新评估？

**判断逻辑**：
- 已有评分数据 + 未要求重新评估 → 流程结束，输出"已有评分数据，未重新评分"
- 无评分数据 或 用户要求重新评估 → 继续执行

### 2. 工时预估

以**高级工程师**的角色对需求进行工时预估，考虑以下因素：

#### 评估维度

| 维度 | 考量内容 |
|------|---------|
| 功能复杂度 | 核心功能点数量、业务规则复杂度、数据处理逻辑 |
| 技术难度 | 是否涉及新技术、外部系统集成数量、性能要求 |
| 测试工作量 | 单元测试、集成测试、端到端测试的覆盖需求 |
| 联调成本 | 外部系统接口联调、前后端联调 |
| 风险缓冲 | 技术不确定性、需求变更风险 |

#### 工时估算标准

- 基准：1 位高级工程师独立完成
- 单位：人时（小时）
- 包含：开发 + 自测 + 代码审查 + 文档
- 不包含：需求澄清、方案评审、上线部署

#### 更新文档

将工时预估结果更新到需求文档的"基本信息"章节：

```yaml
预估工时: XX 人时
```

同时更新"人力与工时"章节（如存在）：

```markdown
* 全量工作1位高级工程师完成工时预估：XX 人时
* 全量工作1位中级工程师完成工时预估：XX 人时（通常为高级工程师的 1.3-1.5 倍）
```

### 3. RICE 价值规模评分

参照 `../references/rice-scoring-standard.md` 中的 RICE 模型进行价值规模评分。

#### RICE 公式

**RICE Score = (Reach × Impact × Confidence) / Effort**

#### 参数评估

**Reach（触达）**：
- 评估该需求影响的用户数量或事件数
- 影响 100% 用户记为 100，50% 记为 50，依此类推
- 内部工具类需求按实际使用人数估算

**Impact（影响程度）**：
根据需求的业务价值和优先级确定：

| 等级 | 分值 | 适用场景 |
|------|------|---------|
| 解决 Bug | 10 | 修复线上问题、安全漏洞 |
| 高优先级 | 8 | 核心功能、用户强烈需要 |
| 中优先级 | 5 | 重要功能改进 |
| 低优先级 | 2 | 一般功能增强 |
| Nice to Have | 1 | 锦上添花的改进 |

**Confidence（信心指数）**：
根据需求的明确程度确定：

| 等级 | 百分比 | 适用场景 |
|------|--------|---------|
| 高信心 | 100% | 需求文档完善、技术方案明确、有用户调研支持 |
| 中信心 | 75% | 需求基本明确、用户体验/性能优化类 |
| 低信心 | 50% | 需求不够明确、属于探索性功能 |
| 极低信心 | 25% | 纯猜测用户需要 |

**Effort（开发工作量）**：
- 使用步骤 2 中预估的工时
- 按每天 8 小时折算为人天
- 例如：预估 40 人时 → Effort = 5 人天

#### 计算并记录

计算 RICE 得分并更新到需求文档的"基本信息"章节：

```yaml
价值规模: [RICE得分]（Reach=[R], Impact=[I], Confidence=[C]%, Effort=[E]人天）
```

### 4. 保存更新

将更新后的需求文档保存到原文件路径。如果文档来源于 TAPD 而非本地文件，则保存到
`docs/reqs/` 目录下。

> **跨平台提示**：使用 Agent 内置的 `write_to_file` 工具保存文件，避免依赖
> 特定操作系统的 Shell 命令。

### 5. 输出评分结果

输出结构化的评分结果，供主 skill 继续处理：

```markdown
## 评分结果

**需求名称**：[名称]
**预估工时**：XX 人时（XX 人天）
**RICE 评分**：XX
  - Reach: XX
  - Impact: XX
  - Confidence: XX%
  - Effort: XX 人天
```

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 需求文档内容不足以评估 | 标注"信息不足"，使用保守估算并说明假设 |
| 文件保存失败 | 将评分结果输出到控制台，提示用户手动更新 |
