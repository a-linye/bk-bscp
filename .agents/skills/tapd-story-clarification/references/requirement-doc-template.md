# 标准化需求文档模板

本模板用于需求澄清后生成规范化的 Markdown 需求文档。Agent 应根据澄清结果填写
各章节，信息充分的详细填写，信息不足的标注"待确认"。

---

## 模板使用说明

### 简单需求精简版

对于非常简单的需求（纯配置变更、文案修改等），只需填写：
- 基本信息
- 需求背景（简要）
- 功能需求（核心功能点）
- 验收标准
- 原需求描述

### 复杂需求完整版

对于复杂需求（涉及外部系统、多角色、性能要求高），所有章节必须填写，重点关注：
- 外部依赖与集成（接口契约必须明确）
- 业务规则（边界条件必须覆盖）
- 非功能需求（指标必须量化）

### 文档质量标准

- 避免使用"较快""较好""灵活"等模糊词汇
- 所有性能指标必须使用可量化的数字
- 验收标准必须使用 Given-When-Then 格式
- 外部接口必须有请求/响应示例
- 原始需求描述必须完整保留
